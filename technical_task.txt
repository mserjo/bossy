Бонусна система в межах своєї групи (нагороди/бонуси/штрафи за виконання завдань та за події).
Робоча Назва: Kudos
Запасні варіанти: Bossy/BonGru/GruBon/Incentive/Laudis/Virtus (або запропонуй свій варіант, можливо на латині).

Основні технології (можеш додати ще необхідні бібліотеки та технології):
backend: Python 3.13, pytest, FastAPI, PostgreSQL, Pydantic, alembic
frontend: Flutter, flutter_test

Вимоги до системи:
*) Асинхронність
*) Підтримка online та offline режимуІ
*) Кросс-платформність (Android, iOS, Windows, Web, Ubuntu)
*) Аутентифікація - JWT токени + refresh токени
*) Нотифікації - внутрішні в додатку
*) Логування backend та frontend
*) Використання docker та docker-compose
*) Redis для черг/кешування
*) Підтримка різних мов (одразу переклади для англійської, української) і UI, і контент, і помилки, і сповіщення
*) Різні теми оформлення frontend
*) backend - Індекси для поліпшення продуктивності
*) backend - Винесення в базові класи спільні поля/методи/правила
*) backend - Валідація даних у схемах (Pydantic)
*) підтримка завантаження файлів (аватари, іконки груп/нагород), локально
*) покриття тестами всього коду (backend, frontend)
*) колекція Postman для тестування API
*) Health Check API
*) API для зовнішніх систем
*) Можливість синхронізації з Google Calendar, Outlook та інші календарі
*) Можливість сповіщень через месенджери (Telegram/Viber/Slack/Teams та інші)

Основні об'єкти (кожна окремою моделью):
- налаштування системи (для суперюзера): загальні налаштування самої системи, можливість перегляду та налаштувань груп та користувачів
- моніторинг системи (для суперюзера)
- налаштування групи (для адміна групи)
- (довідник) статуси
- (довідник) ролі користувачів
- (довідник) типи користувачів
- (довідник) типи груп
- (довідник) типи завдань
- (довідник) типи бонусів (обирається і налаштовується в налаштуваннях групи)
- (довідник) календарів - налаштовує тільки суперюзер
- (довідник) месенджерів - налаштовує тільки суперюзер
- користувачі
- групи
- завдання/події (з поміткою "постійне", "одноразове", чи з терміном виконання)
- бонуси за завдання
- рахунки користувачів, на якому накопичуються бонуси, або борг
- нагороди (за бонуси)

Основні дії:
*) Користувач (усі ролі) - авторизація , реєстрація за поштою/телефоном та інше, нагадування пароль, відновлення пароля
*) Користувач (адмін групи) - запрошення по лінку, підтвердження нового користувача
*) Налаштування всієї системи; доступів для ролей; довідників; груп - тільки суперюзер
*) Налаштування групи (наприклад, назва групи, назва валюти бонусів, бонуси цілі чи дробові, максимально розмір боргу, і т.д.) - тільки суперюзер/адмін групи
*) (довідник) Статуси - перегляд список/детально - усі ролі; усі дії - тільки суперюзер
*) (довідник) Типи користувачів - перегляд список/детально - усі ролі; усі дії - тільки суперюзер
*) (довідник) Типи груп - перегляд список/детально - усі ролі; усі дії - тільки суперюзер
*) (довідник) Типи завдань - перегляд список/детально - усі ролі; усі дії - тільки суперюзер
*) (довідник) Типи бонусів - перегляд список/детально - усі ролі; усі дії - тільки суперюзер
*) Користувачі - перегляд список/детально та "виписки" бонусів та нагород - усі ролі в рамках групи та доступів; усі дії - тільки суперюзер/адмін групи
*) Групи - перегляд список/детально, покинути групу, звіти - усі ролі в рамках групи та доступів; усі дії - тільки суперюзер
*) Адмін групи не може покинути групу, якщо він один адмін в цій групі
*) Завдання - перегляд список/детально, статус "виконано" - усі ролі в рамках групи та доступів; усі дії - тільки суперюзер/адмін групи
*) Події - перегляд список/детально, статус "виконано" - усі ролі в рамках групи та доступів; усі дії - тільки суперюзер/адмін групи
*) Нагороди - перегляд список/детально, отримати за бонуси - усі ролі в рамках групи та доступів; усі дії - тільки суперюзер/адмін групи
*) Рахунок - створюється автоматично для кожного користувача; кожен бачить свій рахунок, виписку по рахунку;
*) суперюзер/адмін групи - бачать усі рахунки згруповані по групах та в рамках своїх доступів
*) Після авторизації, користувачу виводиться для вибору список груп до яких у нього є доступ (з вказанням ролі) або створення нової групи
*) Користувач, який створив групу автоматично стає адміном групи
*) Адмін групи можуть додавати користувачів з ролями "користувач"/"адмін", або відправляти запрошення на приєднання (посилання/код унікальний в рамках групи)
*) Адмін також може виконувати задачі/події і також отримувати бонуси чи штрафи
*) Користувач може брати (або адмін назначати користувачу) доступні завдання в роботу і ставити помітку "виконано" на своїх завданнях
*) Одне завдання може виконуватися кількома користувачами (і залежить від типу завдання: 1) виконав перший і завдання закрите; 2) кожен користувач може може виконати завдання і отримати бонуси за виконання)
*) Завдання можуть бути одноразові (після виконання статус "Виконане" або "Закрите"), а також можуть бути постійні завдання (статус змінює тільки адмін, коли задача не актуальна)
*) Коли користувач позначає, що він виконав завдання, то адміністратору групи приходить сповіщення, що треба перевірити та підтвердити завдання щоб зарахувалися бонуси
*) Коли адміністратор групи підтверджує/відхиляє виконання завдання яке користувач помітив як "виконано", то користувачу приходить сповіщення
*) Користувачу приходять сповіщення про усі рухи по рахунку та боргу (наданому кредиту); коли час скоро закінчується строк завдання
*) Користувач сам обирає в своєму профілі які саме і куди будуть приходити сповіщення
*) Суперадмін налаштовує в своєму профілі які саме і куди будуть приходити сповіщення (по роботі системи, про помилки)
*) Завдання/події можуть бути з негативними бонусами (штрафи). Наприклад, подія на постійній основі - "Запізнився на роботу", "Не прибрав в кімнаті"
*) Завдання можуть бути "обов'язковими", тобто за невиконання завдання в термін - штраф
*) Можливість налаштувати додаткові бонуси за послідовне виконання завдань без пропусків
*) Користувачі можуть залишати відгуки на завдання та ставити рейтинги (налаштовується в групі)
*) Користувачам можуть надаватися рівні, бейджі, прогрес-бари за виконану кількість завдань/подій (налаштовується в групі)
*) Рейтинги користувачів за накопиченими/заробленими бонусами в групі

Ініціалізації початкових даних (може виконувати суперадмін по кнопці):
- запускається разом із системою та перевіряє чи створені/заповнені довідники
- (довідник) ролі користувачів - суперюзер (вся система), адмін (група), користувач (група);
- (довідник) тип користувача - суперюзер, адмін, користувач, бот;
- (довідник) тип групи - сім'я, відділ, організація;
- (довідник) тип завдання - звичайне завдання, складне завдання, подія, штраф;
- (довідник) тип бонусів - бонус.іі
Також при ініціалізації треба створювати користувачів odin (суперадмін), shadow (системний бот-адмін, під ним будуть виконуватися фонові задачі, крон, та інше)

Предзаповнення довідників даними:
*) ролі користувачів - суперюзер (доступна вся система), адмін (доступна група), користувач (група)
*) тип користувача - суперюзер, адмін, користувач, бот
*) тип групи - сім'я, відділ, організація
*) тип завдання - звичайне завдання, складне завдання, подія
*) тип бонусів - бонус
