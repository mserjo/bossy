Потрібно написати код для такої системи (весь код повинен бути оптимізований та з максимально детальними коментарями):

Бонусна система в межах своєї групи (нагороди/бонуси/штрафи за виконання завдань та за події).
Назва: Bossy або Kudos (або запропонуй свій варіант).

Основні технології (можеш додати ще необхідні бібліотеки та технології):
backend: Python 3.13, pytest, FastAPI, PostgreSQL, Pydantic, alembic
frontend: Flutter, flutter_test

Вимоги до системи:
*) Асинхронність
*) Підтримка online та offline режимуІ
*) Кросс-платформність (Android, iOS, Windows, Web, Ubuntu)
*) Аутентифікація - JWT токени + refresh токени
*) Нотифікації - внутрішні в додатку
*) Логування backend та frontend
*) Використання docker та docker-compose
*) Redis для черг або кешування
*) Підтримка різних мов (одразу переклади для англійської, української) і UI, і контент, і помилки, і сповіщення
*) Різні теми оформлення frontend
*) backend - Індекси для поліпшення продуктивності
*) backend - Винесення в базові класи спільні поля/методи/правила
*) backend - Валідація даних у схемах (Pydantic)
*) підтримка завантаження файлів (аватари, іконки груп/нагород), локально
*) покриття тестами всього коду (backend, frontend)
*) колекція Postman для тестування API
*) Health Check API

Основні об'єкти (кожна окремою моделью):
- налаштування системи (для суперюзера): загальні налаштування самої системи, можливість перегляду та налаштувань груп та користувачів
- моніторинг системи (для суперюзера)
- налаштування групи (для адміна групи)
- тип користувача
- ролі користувачів
- користувач
- тип групи
- група
- тип завдання
- завдання/подія (з поміткою "постійне", "одноразове", чи з терміном виконання)
- тип бонусів (обирається і налаштовується в налаштуваннях групи)
- бонуси за завдання
- рахунок користувача, на якому накопичуються бонуси, або борг
- нагороди (за бонуси)

Основні дії:
*) Авторизація, реєстрація, нагадування пароль, відновлення пароля, запрошення по лінку, підтвердження нового користувача адміністратором групи
*) Тип користувача - перегляд список/детально, створення, редагування, видалення
*) Користувач - перегляд список/детально, створення, редагування, видалення, перегляд "виписки" бонусів за завдання, перегляд "виписки" нагород за завдання, можливість надання бонусів в борг
*) Тип групи - перегляд список/детально, створення, редагування, видалення
*) Група - перегляд список/детально, створення, редагування, видалення
*) Тип завдання - перегляд список/детально, створення, редагування, видалення
*) Завдання - перегляд список/детально, створення, редагування, видалення, помітити як виконане, повернути на виконання
*) Тип бонусів - перегляд список/детально, створення, редагування, видалення
*) Нагорода - перегляд список/детально, створення, редагування, видалення
*) Рейтинги користувачів за накопиченими/заробленими бонусами в групі
*) Користувач реєструється в системі за поштою/телефоном/google аккаунтом та інше
*) Після авторизації, користувачу виводиться для вибору список груп до яких у нього є доступ (з вказанням ролі) або створення нової групи
*) Користувач, який створив групу автоматично стає адміном групи і може додавати адмінів в цю групу
*) Адмін групи можуть додавати користувачів з такими правами "адмін" або "користувач", або відправляти запрошення на приєднання (посилання/код запрошення не повторюються)
*) Адмін також може виконувати задачі/події і також отримувати бонуси чи штрафи
*) В межах своєї групи, адмін може виконувати над об'єктами усі дії (а також налаштування групи, наприклад, назва групи, назва валюти бонусів, бонуси цілі чи дробові, максимально дозволений розмір боргу, і т.д.)
*) В межах своєї групи, користувач може переглядати інформацію та звіти (крім налаштувань групи)
*) Користувач може брати (або адмін назначати користувачу) доступні завдання в роботу і ставити помітку "виконано" на своїх завданнях
*) Одне завдання може виконуватися кількома користувачами (і залежить від типу завдання: 1) виконав перший і завдання закрите; 2) кожен користувач може може виконати завдання і отримати бонуси за виконання)
*) Завдання можуть бути одноразові (після виконання статус "Виконане" або "Закрите"), а також можуть бути постійні завдання (статус змінює тільки адмін, коли задача не актуальна)
*) Коли користувач позначає, що він виконав завдання, то адміністратору групи приходить сповіщення, що треба перевірити та підтвердити завдання щоб зарахувалися бонуси
*) Коли адміністратор групи підтверджує/відхиляє виконання завдання яке користувач помітив як "виконано", то користувачу приходить сповіщення
*) Користувачу приходять сповіщення про усі рухи по рахунку та боргу (наданому кредиту)
*) Завдання/події можуть бути з негативними бонусами (штрафи). Наприклад, подія на постійній основі - "Запізнився на роботу", "Не прибрав в кімнаті"
*) Завдання можуть бути "обов'язковими", тобто за невиконання завдання в термін - штраф
*) Користувачі можуть залишати відгуки на завдання та ставити рейтинги (налаштовується в групі)
*) Користувачам можуть надаватися рівні, бейджі, прогрес-бари за виконану кількість завдань/подій (налаштовується в групі)

Ініціалізації початкових даних (може виконувати суперадмін по кнопці):
- запускається разом із системою та перевіряє чи створені/заповнені довідники
- початкові дані: ролі користувачів - суперюзер (вся система), адмін (група), користувач (група);
- тип користувача - суперюзер, адмін, користувач, бот;
- тип групи - сім'я, відділ, організація;
- тип завдання - звичайне завдання, складне завдання, подія, штраф;
- тип бонусів - бонус.
Також при ініціалізації треба створювати користувачів root (суперадмін), odin (суперадмін), shadow (системний бот-адмін, під ним будуть виконуватися фонові задачі, крон, та інше)

Предзаповнення даними:
*) ролі користувачів - суперюзер (доступна вся система), адмін (доступна група), користувач (група)
*) тип користувача - суперюзер, адмін, користувач, бот
*) тип групи - сім'я, відділ, організація
*) тип завдання - звичайне завдання, складне завдання, подія
*) тип бонусів - бонус
